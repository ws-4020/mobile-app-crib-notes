React Native開発環境の構築手順です。環境構築を進める前にツールのインストール状況を確認してください。該当バージョンのツールがすでにインストールされている場合、再インストールは不要です。それ以外の場合は再インストール（アップデート）してください。

## インストール状況とバージョンの確認

ツールのインストール状況とバージョンの確認手順は次の通りです。

### Node.js

コマンドラインかターミナルで`node -v`を実行してください。バージョン `v10.x.x` 以降であればOKです。

```console
$ node -v
v14.15.1
```

エラーが表示された場合、またはバージョンが`v9.x.x`以下の場合はインストールが必要です。

### Java Development Kit

コマンドラインかターミナルで`java -version`を実行してください。バージョン `1.8.x` 以降であればOKです。

```console
$ java -version
openjdk version "1.8.0_275"
OpenJDK Runtime Environment (AdoptOpen_JDK)(build 1.8.0_275-b01)
OpenJDK 64-Bit Server VM (AdoptOpenJDK)(build 25.275-b01, mixed mode)
```

上記は`OpenJDK`をインストールしている場合に出力されるメッセージです。他のJDKと比較して一部メッセージ内容に相違はありますが、バージョンの確認箇所は、`openjdk version "1.8.0_275"`の`"1.8.0_275"`（バージョン）部分になります。

エラーが表示された場合、またはバージョンが`1.7.x`以下の場合はインストールが必要です。

### Android Studio

Android Studioがインストールされていることを確認してください。インストールされていない場合、インストールが必要です。
インストールされている場合、Android SDKのインストール状況も確認してください。確認方法は、[Android SDK](#android-sdk)を参照してください。

### Android SDK

Android開発はAndroid SDKのツール類を使用します。次の手順にそってツールが利用可能な状態かを確認してください。
SDK Managerにてツールの状況を次の手順にしたがって確認してください。

1. Android Studioを起動
2. `Configure > SDK Manager > Appearance & Behavior > System Setting > Android SDK`まで移動
3. `SDK Tools`タブを選択
4. `Android SDK Platform-Tools`がインストール状態で表示されていることを確認

Android SDK Platform-Toolsがインストールされていない場合はインストールしてください。

つづいて環境変数の`PATH`に以下のパスが含まれていることを確認してください。

### react-native-cli

コマンドラインかターミナルで`npm ls -g --depth=0` を実行して`react-native-cli` がインストール**されていないこと**を確認してください。

:::caution
`react-native-cli`がインストールされていると、アプリ実行や新規プロジェクトの作成に失敗します。
詳細については、[react-native-cliについて](../../troubleshooting/react-native-cli-uninstall.md)を参照してください。
:::

### Xcode（macOSのみ）

Xcodeがインストールされていることを確認してください。インストールされている場合、Xcodeを起動してください。起動後のウィンドウにバージョンが表示されています。`12.x` 以降であればOKです。

Xcode表示例：`Version 12.3(12C33)`

Xcodeがインストールされていない場合、またはバージョンが`11.x`以下の場合はインストールが必要です。

### Watchman（macOSのみ）

ターミナルで`watchman -v`を実行してください。

```console
$ watchman -v
4.9.0
```

エラーが表示された場合、インストールが必要です。バージョンに関して特に指定はありません。

### CocoaPods（macOSのみ）

ターミナルで`pod --version`を実行してください。

```console
$ pod --version
1.10.1
```

エラーが表示された場合、インストールが必要です。バージョンに関して特に指定はありません。

## 環境構築

[React Nativeの公式ドキュメント](https://reactnative.dev/docs/environment-setup)を参照して、不足しているツールをインストールしてください。公式ドキュメントにはWindowsとmacOS両方の説明があります。環境に応じて参照する内容を選択してください。

:::info
[React Nativeの公式ドキュメント](https://reactnative.dev/docs/environment-setup)を参照する際は、**React Native CLI Quickstart**タブを選択して、参照してください。
:::

## 補足事項

### 開発に利用するエディタ

開発時に使用するエディタをインストールします。エディタは使い慣れたものなら何でもいいです。

特にこだわりがなければ`Visual Studio Code`をお勧めします。[公式サイト](https://azure.microsoft.com/ja-jp/products/visual-studio-code/)の案内に沿って、インストールしてください。

### プロキシ設定

社内ネットワークなどプロキシ環境下で開発環境を構築する場合、プロキシに関する設定が必要です。次の手順にしたがい設定してください。

:::caution
以下では、プロキシに`http://proxy.example.com:8080`を設定する例を紹介しています。実際のプロキシに合わせて設定を変更してください。
:::

#### 環境変数

環境構築および、実行時等に使用する一部ツール（npmやCocoaPods等）のプロキシ設定です。次のコマンドで環境変数を設定してください。

<TabItem value="mac">

`~/.bash_profile`もしくは`~/.zprofile`へ、以下の内容を追記してください。

```console
export http_proxy=http://proxy.example.com:8080
export https_proxy=http://proxy.example.com:8080
export no_proxy=localhost,127.0.0.1
```

すでに開いているシェルに変更を反映するには、`source ~/.bash_profile`もしくは`source ~/.zprofile`を実行して設定を読み込んでください。

:::info
macOS Catalina (10.15) からデフォルトのシェルがZ shell (zsh)に変更されています。

使用しているシェルは`echo $SHELL`で確認できるので、bashだった場合は`~/.bash_profile`に、zshだった場合は`~/.zprofile`に設定を追加してください。
:::

</TabItem>
<TabItem value="win">

```console
setx http_proxy http://proxy.example.com:8080
setx https_proxy http://proxy.example.com:8080
setx no_proxy localhost,127.0.0.1
```

</TabItem>
</Tabs>

#### Gradle

Gradleで利用するプロキシは`~/.gradle/gradle.properties`に設定します。詳細は[Build Environment - Gradle](https://docs.gradle.org/current/userguide/build_environment.html#sec:accessing_the_web_via_a_proxy)を参照してください。

```console
systemProp.http.proxyHost=proxy.example.com
systemProp.http.proxyPort=8080
systemProp.https.proxyHost=proxy.example.com
systemProp.https.proxyPort=8080
```
:::info
`~/.gradle/gradle.properties`が存在しない場合、新規作成してください。

`~`は、Windowsであれば`%USERPROFILE`、macOSであれば`$HOME`です。
:::
