title: エラーハンドリング
---

バックエンド実行結果のハンドリング方法を説明します。
エラーの表示方法は [Alert](https://reactnative.dev/docs/alert) を利用します。

## エラーハンドリングの仕様

ハンズオンで作成するアプリのエラーハンドリング仕様を説明する。

- クライアントエラー（400系）
ポップアップまたは、画面表示して、利用者に入力誤りであることを促す。
- サーバエラー（500系）
メッセージ（ポップアップ）を表示して、システムエラーが発生していることを促す。
- Networkエラー
メッセージ（ポップアップ）を表示して、システムエラーが発生していることを促す。

実際はプロジェクトにて設計を行いハンドリング仕様を決定するので、
それぞれのプロジェクトにて設計した仕様で実装すること。

:::caution
OpenAPIドキュメントとのI/F不一致が原因となるエラーはハンドリングしない。
そもそも、開発時のバグであり、ハンドリング対象ではないため
:::

非同期処理の結果を受けて、それぞれのHTTPステータスを判断して、メッセージ（ポップアップ）表示や
画面出力処理を行う方法を説明する。
コード紹介と追加部分の説明。

## 処理中のエラーハンドリング方法

非同期処理中に発生したエラーを取得して、メッセージ（ポップアップ）表示をする方法を説明する。
コード紹介と追加部分の説明。

## 動作確認

以下それぞれの異常系の動作確認を行う。

- クライアントエラー（400系）
PUT：404。
- サーバエラー（500系）
GET、POST、PUT、DELETE。
- Networkエラー
GET、POST、PUT、DELETE。
これは、ローカル環境のみで確認する。

それぞれの再現方法（エラーの発生）も説明する。
