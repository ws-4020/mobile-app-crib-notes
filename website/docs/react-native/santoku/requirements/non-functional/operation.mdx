>情報システムの運用時間、運用監視、障害復旧、その他の運用管理方針、運用環境等に関する要件を記載する。
[guideline]:<https://cio.go.jp/sites/default/files/uploads/documents/hyoujun_guideline_20210330.pdf>

本項では、このアプリの運用に関する要件として、以下の事項を定義します。

## 要件内容

- ログ集積の実施について
- ストア上での評価の扱いについて

### ログ集積の実施について

このアプリでは以下の外部サービスと連携して利用状況やアプリのクラッシュログを集積し、機能改善や問題の解決に使用します。

- [Firebase Crashlytics][Crashlytics]
- [Google Analytics for Firebase][Analytics]

Webシステムのようにサーバにシステムを構築する場合、障害などが発生した際に原因を特定するために必要となるログを集積します。
ですが個人が携行するスマートフォン端末においては端末内のアプリでログを出力してもその内容を直接参照できません。
そこで[Firebase Crashlytics][Crashlytics]や[Google Analytics for Firebase][Analytics]を利用して端末からレポートの送信し、改善や問題の解決のために使用します。
なおこの時送信される情報は匿名化がされており特定のユーザにリンクされていません。
またログを収集するにあたり、ユーザに対し収集する内容と利用目的に同意してもらうように利用規約に記載します。

[Crashlytics]:https://firebase.google.com/products/crashlytics?hl=ja
[Analytics]:https://firebase.google.com/products/analytics?hl=ja

### ストア上での評価の扱いについて

アプリストア上での評価の取り扱いはモバイルアプリケーションの発注者が管理するものとします。
アプリストア上の評価を鑑みてモバイルアプリケーションの信頼性改善や追加機能開発、改善といった依頼をするのが発注者側であることから管理は発注者で行うものとしています。
