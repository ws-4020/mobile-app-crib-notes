title: テストに関する事項
---

テストに関する事項は政府公開の[デジタル・ガバメント推進標準ガイドライン][guideline]に照らし合わせると以下となります。
>情報システムの設計から運用開始に至るまでの全てのテストについて、テストの種類、目的、内容、実施者、合否判断基準、テスト実施環境などを記載する。
[guideline]:<https://cio.go.jp/sites/default/files/uploads/documents/hyoujun_guideline_20210330.pdf>

本項では、サンプルアプリケーションのテストに関する要求として、以下の事項を定義します。

## 要件内容

- 実施タイミング
- 検証観点
- 実施方法

## 実施タイミング

このアプリでは、以下工程のテストを実施します。

| 工程 | 概要 |
| :--- | :--- |
| 製造／単体テスト | 機能単位でのレイアウト（表示崩れなど）と動作を検証する |
| 結合テスト | モバイルアプリケーションの機能と連携するサービスを含んでの網羅的な動作と性能を検証する |
| システムテスト | 要求した機能および、性能を満たしていること検証する |

## 検証観点

テスト検証観点として、大きく以下のカテゴリに分類して、テストします。
それぞれのカテゴリをテストするタイミングとテスト内容に関する詳細は、テスト計画にて決定します。

| カテゴリ | 検証概要 |
| :--- | :--- |
| [機能テスト](#機能テスト) | モバイルアプリケーションの機能が動作することを検証する |
| [構成テスト（OSバリエーション）](#構成テストosバリエーション) | 各種OSバージョン毎にモバイルアプリケーションの機能が動作することを検証する |
| [構成テスト（レイアウトバリエーション）](#構成テストレイアウトバリエーション) | 異なる画面解像度毎のレイアウト崩れがないことを検証する |
| [構成テスト（ネイティブ機能バリエーション）](#構成テストネイティブ機能バリエーション) | このアプリがサポートしているネイティブ機能（カメラ、NFC、生体認証など）を検証する |
| [性能テスト](#性能テスト) | モバイルアプリケーションのレスポンス遅延がないことと、メモリリークが発生しないことを検証する |
| [セキュリティテスト](#セキュリティテスト) | モバイルアプリケーションのセキュリティ品質が確保できているかを検証する |

### 機能テスト

要求、仕様通りに機能が動作することと、レイアウト崩れなどがなく表示されることを検証します。

### 構成テスト（OSバリエーション）

AndroidとiOS(iPadOS)、それぞれでOSバージョンを準備して、OSバージョン毎に動作を検証します。
このアプリがサポートするOSバージョンの範囲は、[配信対象OSバージョン](usability#配信対象osバージョン)に従って、
テスト対象のOSバリエーションをAndroid、iOS（iPadOS）それぞれで選定します。

### 構成テスト（レイアウトバリエーション）

それぞれの画面解像度を準備して、レイアウト崩れがないかなどを検証します。
画面解像度の選定基準を以下に定義します。

- Androidの選定基準

  Androidは複数のメーカーがそれぞれ異なる形状の機種を扱っており、画面解像度のバリエーションが多く存在します。
  その為、どのメーカーのシェア（利用度合い）が高いかを開発時点から4年前までの期間の中から特定します。
  特定したメーカーとAndroidの開発元であるGoogleを合わせて、以下に従って、画面解像度を選定します。

  | 選定基準 | 選定内容 |
  | :--- | :--- |
  | アスペクト比率 | メーカー毎にローエンド&ミドルレンジ機種を選定して、アスペクト比率毎の画面解像度を選定（※1） |
  | パンチホール機種 | パンチホール有の機種の中で最大の画面解像度を選定 |

  （※1）アスペクト比で重複するものは、端末性能の低い方に統一する（端末性能も同じ場合、発売日が古い機種に統一する）

- iOS（iPadOS）の選定基準

  シェア（利用度合い）の高い機種を特定して、その機種の画面解像度を選定します。
  その他に配信対象OSバージョンがサポートする機種の中から以下に従って、選定します。

  | 選定基準 | 選定内容 |
  | :--- | :--- |
  | 画面解像度 | 最大と最小の画面解像度を選定 |
  | ノッチ | ノッチ有の機種の中で最大の画面解像度を選定 |

### 構成テスト（ネイティブ機能バリエーション）

生体認証で最も一般的な顔認証と指紋認証をテスト対象とします。
生体認証機能を踏まえてテストに必要な機種を準備します。

### 性能テスト

このアプリの機能をテストして、レスポンス遅延がないことと、メモリリークが発生しないことを検証します。

### セキュリティテスト

このアプリにて、不当な行為に対する脆弱性がないかを検証します。

## 実施方法

テストは、主に手動と自動の2種類でテストします。それぞれの実施内容は以下になります。

| テスト方法 | テスト概要 |
| :--- | :--- |
| 手動 | エミュレータ／シミュレータと実機を使用して、レイアウトに関する部分と性能を検証する |
| 自動 | ロジック部分の網羅条件を全て満たしていることを検証する |
