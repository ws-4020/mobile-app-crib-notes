## ディレクトリ名

ディレクトリ名は、ケバブケース記法とします。

以下は本規約に従った命名例です。

```console
src
├── features
│   ├── qa-home
```

## assetファイル名

画像などのassetファイルは、ケバブケース記法とします。

以下は本規約に従った命名例です。

```console
beginner-mark.png
```

## JavaScript/TypeScript

### ファイル名

JavaScript/TypeScriptのファイル名は、`export`しているものと同じ名前にしてください。

[JavaScript/TypeScriptのコーディング規約 - Moduleの単位](./js-ts-coding-rules.mdx#moduleの単位)に記載の通り、基本的に`Module`は1つの`export`を持ちます。
例外的に複数の`export`がある場合は、その`Module`を表す代表的な`export`対象の名前をファイル名にしてください。

また、`Barrel`に関しては、`index.[js/ts]`というファイル名にしてください。

## JSX/TSX

<!-- markdownlint-disable MD024 -->

### ファイル名

<!-- markdownlint-enable MD024 -->

JSX/TSXのファイル名は、`export`しているコンポーネントと同じ名前にしてください。

JSX/TSXも[JavaScript/TypeScriptのコーディング規約 - Moduleの単位](./js-ts-coding-rules.mdx#moduleの単位)と同様に、基本的に`Module`は1つの`export`を持ちます。
例外的に複数の`export`がある場合でも、`export`するコンポーネントは1つとし、そのコンポーネント名をファイル名にしてください。

### コンポーネント名

コンポーネント名は、[JavaScript/TypeScriptのコーディング規約 - 変数](./js-ts-coding-rules.mdx#変数-)に準拠します。ただし、先頭文字は大文字としてください。

以下は本規約に従った命名例です。

```tsx
const HomePage: React.FC = () => { // (名詞)
// ...
};
const SelectableSheet: React.FC = () => { // (形容詞) + (名詞)
// ...
};
const TotalNumOfEvent: React.FC = () => { // (形容詞) + (名詞) + (形容詞句)
// ...
};
```
