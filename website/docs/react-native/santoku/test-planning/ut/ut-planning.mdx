## テスト実施内容

テストは手動（打鍵）と自動の2種類を実施します。
それぞれのテスト方法毎の確認内容は以下のとおりです。

### 手動（打鍵）テストの実施内容

- 開発した画面または機能に関連する設計書と比較してレイアウトやイベントの動作を検証する
- API連携部分は、モックサーバと接続して、動作を検証する
- テスト観点表をベースにテスト対象の画面または機能を検証する観点を選定して、単体テスト仕様書を作成して動作を検証する

### 自動テスト

- テスト観点表をベースにテスト対象の画面または機能を検証する観点を選定して、単体テスト仕様書とテストコードを作成して動作を検証する
- テストコードは、C0/C1カバレッジが100%とるように作成する

## エビデンス取得方針

- エビデンスは基本取得しない。テストケースに対するテスト結果を単体テスト仕様書に記載する
- 自動テストの結果を確認して、カバレッジが100%になっていることを検証する。カバレッジが100%にならない場合は、その理由を単体テスト仕様書へ記載する

## 検証方針

- 担当者はテスト結果を確認して単体テスト仕様書のテストケースを網羅していることと、カバレッジが100%となっていることを確認する。カバレッジが100%となっていない場合、未達理由を検証して妥当であることを確認する
- 再鑑者が単体テスト仕様書の内容について、テスト結果が妥当であることを確認する

## テスト環境

単体テストは、開発用PCのローカル環境で実施する。エミュレータ／シミュレータまたは実機で使用する機種とOSバージョンのバリエーションは、
[単体テスト環境](../all/test-environment#単体テスト環境)で、定義したバリエーションでテストを実施する。

## 事前準備

テスト実施前に以下のことを準備する。

### テストデータ

テスト実施に必要なデータパターンを洗い出し、テストケースを網羅できるテストデータを作成する。
