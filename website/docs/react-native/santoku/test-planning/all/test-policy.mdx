title: テスト方針
---

このテスト方針は、サンプルアプリケーション開発における全体のテスト方針を取りまとめたものです。

## 品質方針

品質評価基準(mearge後にリンクにする)に従って、以下の成果物に対して品質分析を実施して品質評価を行います。

- 設計書やテスト仕様書等のドキュメント類
- 開発プログラム
- 単体テストと結合テストのケース密度や不具合検出密度

## 期間・作業内容

全体の開発期間は、`2021/10/1 ～ 2022/03/31`になります。
今回の開発は5つに分けたサイクルで開発を進めていきますので、それぞれのサイクルで作成する単体テスト計画書と結合テスト計画書へスケジュールは記載します。
ここでは、全サイクルを通したテスト工程と作業内容を明記します。

(単体と結合しか実施しない背景を記載しておきたい)

| NO | テスト工程 | 作業内容 |
| --- | :--- | :--- |
| 1 | 単体テスト | 開発した画面（モジュール）の単位で、プログラムの正常と異常の動作検証をする |
| 2 | 結合テスト | 要件定義で検討した機能要件、非機能要件に関する業務運用を想定したシナリオで仕様通りに動作することを検証する |

## 重要度

アーキテクチャ部分の部品開発を考慮して、部品取込みを踏まえた画面及び、機能開発となるような順番としました。
