title: 網羅の方針
---

異なる種類の複数端末とOSバージョンやその他の画面解像度、ネイティブ機能を含めたさまざまな環境で問題なくこのアプリが動作することを検証します。

## テスト対象機種を揃えた網羅

OSバージョンについて、要件を決めた時点で最新であった以下のものを指します。

| OS | 最新バージョン |
| :--- | --- |
| Android | 11 |
| iOS（iPadOS） | 14 |

テスト対象の機種選定は、[配信対象端末](../../requirements/non-functional/usability#配信対象端末)に該当する端末の中からAndroidとiOS（iPadOS）それぞれでシェア（利用度合い）の高い機種を選定します。

### Androidのテスト対象機種

Androidは多くのメーカーが多種多様な機種をリリースしています。なので今回のアプリ開発時点から4年前までの期間の中でシェア（利用度合い）の高いメーカーの中から機種を選定します。
シェアの高いメーカーとGoogleからリリースされているローエンド、ミドルレンジの機種を選定します。

コメント：アスペクト比率で選定する理由が必要。また、画面解像度は最大と最低ものは選定する。

### iOS（iPadOS）のテスト対象機種

iPhoneとiPadそれぞれの機種でシェア（利用度合い）の高い以下の機種をテスト対象端末として準備します。実機で準備が難しいものはシミュレータで準備します。

| OS | 機種名（※1） | アスペクト比率 | 画面解像度 | ノッチ有無 |
| :--- | :--- | :--- | :--- | :--- |
| iOS | iPhone 7 | 9:16 | 750x1334 | 無 |
| iOS | iPhone 12 Pro Max（※2） | 9:19.5 | 2778x1284 | 有 |
| iOS | iPhone SE（第1世代）（※3） | 9:16 | 640×1136 | 無 |
| iPad | iPad（第8世代） | 1640×2360 | 3:4 | 無 |
| iPad | iPad Pro 11インチ（第2世代） | 1668×2388 | 139:199 | 無 |

（※1）OS毎に上から順番にシェアの高い機種を列挙。

（※2）[配信対象端末](../../requirements/non-functional/usability#配信対象端末)がサポートする端末の中からノッチ有で最大の画面解像度。

（※3）[配信対象端末](../../requirements/non-functional/usability#配信対象端末)がサポートする端末の中から最小の画面解像度。

### サポート構成範囲

このアプリがサポートしている端末構成の範囲は以下のとおりです。

| カテゴリ | 区分 | サポート範囲 |
| :--- | :--- | :--- |
| OSバージョン | Android OSバージョン | 8.0 ～ 11 |
| OSバージョン | iOS(iPadOS) | 13 ～ 14 |
| アスペクト比率 | Android（スマートフォン） | 18:9、19:9、20:9、21:9、9:16、19.5:9 |
| アスペクト比率 | Android（タブレット） | 16:10 |
| 画面解像度 | iPhone | 1284×2778、750×1334 |
| 画面解像度 | iPad | 2732x2048、1536×2048 |
| ネイティブ機能 | ― | カメラ、生体認証（指紋認証、顔認証）など |

上記それぞれのサポート範囲の中からこのテストで使用するバージョンや規格を選定基準によって決定します。
また、このアプリのメインターゲットはスマートフォンなので、タブレットでのテストはエミュレータ／シミュレータのみで実施します。

### OSバージョン

OS毎の動作を検証して異なるOSバージョンであっても仕様通りの動作となることを検証します。
テストに関する事項の[テスト対象OSバージョン](../../requirements/non-functional/test-outline#テスト対象osバージョン)のとおり、
サポートしているOSバージョンの中からテスト対象OSバージョンを以下のとおり選定します。

| OSバージョン | 選定理由 |
| :--- | :--- |
| Android 11 | ユーザ使用率の高いOSバージョンのため  |
| Android 10 | ダークモードが利用可能となったバージョンであり、このアプリに影響があるため |
| iOS 14 | ユーザ使用率の高いOSバージョンのため  |
| iOS 13 | シェア3%未満に入らないOSバージョンのため |
| iPadOS 14 | ユーザ使用率の高いOSバージョンのため  |
| iPadOS 13 | シェア3%未満に入らないOSバージョンのため |

### 画面構成

異なる画面解像度でこのアプリを使用し、レイアウト崩れなど表示に関する問題がないことを検証します。
テストに関する事項の[構成テスト（レイアウトバリエーション）](../../requirements/non-functional/test-outline#構成テストレイアウトバリエーション)のとおり、
サポートしているアスペクト比率、画面解像度の中からテスト対象の画面構成を以下のとおり選定します。

| カテゴリ | 選定基準（※1） | 構成 |
| :--- | :--- | :--- |
| Android（スマートフォン） | A | 19:9、19:9、20:9、21:9、9:16 |
| Android（スマートフォン） | B | 19.5:9 |
| Android（タブレット） | A | 16:10 |
| 画面解像度（iPhone） | C | 1284×2778（※2） |
| 画面解像度（iPhone） | D | 750×1334 |
| 画面解像度（iPad） | C | 2732x2048 |
| 画面解像度（iPad） | D | 1536×2048 |

（※1）選定基準は以下のとおりです。

- （A）最多シェアメーカーのローエンドまたは、ミドルレンジの機種で最多シェアのOSバージョン
- （B）パンチホール有の機種の中で最大のアスペクト比率
- （C）[配信対象OSバージョン](../../requirements/non-functional/usability#配信対象osバージョン)の範囲内で最大の画面解像度
- （D）[配信対象OSバージョン](../../requirements/non-functional/usability#配信対象osバージョン)の範囲内で最小の画面解像度

（※2）[配信対象OSバージョン](../../requirements/non-functional/usability#配信対象osバージョン)の範囲内の最大とノッチ有の画面解像度が重複しているため統一して選定。
  
### ネイティブ機能

このアプリが利用するネイティブ機能を利用した部分に問題がないことを検証します。
テストに関する事項の[構成テスト（ネイティブ機能バリエーション）](../../requirements/non-functional/test-outline#構成テストネイティブ機能バリエーション)のとおり、
このアプリに関連する以下のネイティブ機能をテスト対象として選定します。

カメラ、生体認証（指紋認証、顔認証）

### 選定結果のまとめ

Androidまたは、iOS（iPadOS）のテスト対象機種は[Androidのテスト対象機種](#androidのテスト対象機種)と[iOS（iPadOS）のテスト対象機種](#iosipadosのテスト対象機種)のとおりです。
サポート構成範囲は上記のとおり、選定基準を踏まえた結果は以下のとおりです。

| カテゴリ | 区分 | テスト対象範囲 |
| :--- | :--- | :--- |
| OSバージョン | Android OSバージョン | 8.0、10、11 |
| OSバージョン | iOS(iPadOS) | 13、14 |

テスト対象OSバージョン以外の変更はありません。
