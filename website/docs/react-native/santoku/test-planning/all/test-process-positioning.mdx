---
title: テスト工程の位置づけ
---

各テスト工程において、どのような種別のテストを行うのか定義します。
テスト全体で検証する内容を把握し、テストの前後関係を整理します。

## 各テスト工程の目的

- 単体テスト、結合テストで担保すべき内容を記載する（概要レベル）

## 実施するテスト種別

- 単体テスト、結合テストで実施するテスト種別を記載する
  - [参考](https://fintan.jp/?p=45)
  - 各種別のテストを行う目的を記載する
  - テストの実施に優先度や前後関係がある場合は記載する
  - 自動化するテストがあれば、わかるように記載
  - テスト種別ごとに何をテストするのかは個別計画書に記載する。(Fintanでいうところの「網羅の方針」)

## テスト種別ごとの環境要件

### テスト環境

- 単体テスト、結合テストを実施する環境
  - ビルドバリアントやFirebaseどれ使う、とかも
- テスト端末について記載
- テスト全体を通じて、必要なデータの前後関係(例：使い回しできるよね)や、環境をいつ、どう作るかを把握するため

### テストデータ

- 各種別でテストデータの準備が必要な場合は記載
  - テストデータの作成方針を記載
  - テストデータと実データの違いや、テストデータではできないことがあれば記載(個人情報はマスキングするとか)
