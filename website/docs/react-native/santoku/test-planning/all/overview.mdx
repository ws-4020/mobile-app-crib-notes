## テスト方針

[テストに関する事項](../../requirements/non-functional/test-outline)のテスト要件を網羅するテストを実施し、品質を担保します。
テスト範囲はこのアプリのフロントエンド部分を範囲として、開発者のローカル環境で実施します。

## テスト工程定義

このアプリのテストでは、単体テストと結合テストを実施します。
テストに関する事項の[検証観点](../../requirements/non-functional/test-outline#検証観点)で定義したテスト種別をそれぞれのテスト工程で検証します。

:::note info
テストに関する事項の[実施タイミング](../../requirements/non-functional/test-outline#検証観点)では、システムテストもテスト対象としていますが、
このアプリはサンプルであり結合テストまでがテスト範囲のためシステムテストは実施しません。
:::

### 単体テスト

単体テストでは、主に開発した画面または機能単位の動作確認を目的とした検証を実施します。その為、テスト種別として、機能テストをメインにテストします。

| テスト種別 | 目的 |
| :--- | :--- |
| 機能テスト | 画面または機能毎の動作を検証する |

### 結合テスト

結合テストでは、このアプリと連携するサービスを含めた範囲での網羅検証を目的とします。その他に構成テスト、性能テスト、セキュリティテストといったテストを実施して、
このアプリの機能に限らず、複合的な観点での検証を実施します。

| テスト種別 | 目的 |
| :--- | :--- |
| 機能テスト | アプリが連携するサービスを含めた網羅の検証 |
| 構成テスト（OSバリエーション） | テスト対象OSバージョン毎に機能動作を検証（※1） |
| 構成テスト（レイアウトバリエーション） | 選定した画面解像度（パンチホール、ノッチ機種含む）毎にレイアウト崩れがないかの検証（※1） |
| 構成テスト（ネイティブ機能バリエーション） | このアプリに関連するネイティブ機能（カメラ、生体認証など）が正常に利用可能であることを検証（※1） |
| 性能テスト | テスト実施中にレスポンス遅延がないこととその他に性能に問題のある箇所がないことを検証 |
| セキュリティテスト | 外部のセキュリティ診断サービスを利用して、セキュリティ面に問題がないことを検証 |

（※1）テスト対象OSバージョンとテスト対象機種は、[テスト実施端末](test-environment)で選定しています。

## 網羅の方針

### テスト対象機種を揃えた網羅検証

このアプリがサポートしている端末構成を対象にどの環境であっても問題なくアプリが動作することを網羅的に検証します。
OSバージョンについて、要件を決めた時点で最新であった以下のものを指します。

| OS | 最新バージョン |
| :--- | --- |
| Android | 11 |
| iOS（iPadOS） | 14 |

このアプリがサポートしている端末構成の範囲は以下のとおりです。

| カテゴリ | 区分 | サポート範囲 |
| :--- | :--- | :--- |
| OSバージョン | Android OSバージョン | 8.0 ～ 11 |
| OSバージョン | iOS(iPadOS) | 13 ～ 14 |
| アスペクト比率 | Android（スマートフォン） | 19:9、19:9、20:9、21:9、9:16、19.5:9 |
| アスペクト比率 | Android（タブレット） | 16:10 |
| 画面解像度 | iPhone | 1284×2778、750×1334 |
| 画面解像度 | iPad | 2732x2048、1536×2048 |
| ネイティブ機能 | ― | カメラ、生体認証（指紋認証、顔認証） |

上記それぞれのサポート範囲の中からこのテストで使用するバージョンや規格を選定基準によって決定します。
また、このアプリのメインターゲットはスマートフォンなので、タブレットでのテストはエミュレータ／シミュレータのみで実施します。

#### OSバージョン

OS毎の動作を検証して異なるOSバージョンであっても仕様通りの動作となることを検証します。
テストに関する事項の[テスト対象OSバージョン](../../requirements/non-functional/test-outline#テスト対象osバージョン)のとおり、選定したテスト対象OSバージョンは以下です。

| OSバージョン | 選定理由 |
| :--- | :--- |
| Android 11 | ユーザ使用率の高いOSバージョンのため  |
| Android 10 | ダークモードが利用可能となったバージョンであり、このアプリに影響があるため |
| iOS 14 | ユーザ使用率の高いOSバージョンのため  |
| iOS 13 | シェア3%未満に入らないOSバージョンのため |
| iPadOS 14 | ユーザ使用率の高いOSバージョンのため  |
| iPadOS 13 | シェア3%未満に入らないOSバージョンのため |

上記のものをテスト対象OSバージョンとして、決定します。

#### 画面構成

異なる画面解像度でこのアプリを使用し、レイアウト崩れなど表示に関する問題がないことを検証します。
テストに関する事項の[構成テスト（レイアウトバリエーション）](../../requirements/non-functional/test-outline#構成テストレイアウトバリエーション)のとおり、選定した構成は以下です。

| カテゴリ | 選定基準（※1） | 構成 |
| :--- | :--- | :--- |
| Android（スマートフォン） | A | 19:9、19:9、20:9、21:9、9:16 |
| Android（スマートフォン） | B | 19.5:9 |
| Android（タブレット） | A | 16:10 |
| 画面解像度（iPhone） | C | 1284×2778（※2） |
| 画面解像度（iPhone） | D | 750×1334 |
| 画面解像度（iPad） | C | 2732x2048 |
| 画面解像度（iPad） | D | 1536×2048 |

（※1）選定基準は以下のとおりです。

- （A）最多シェアメーカーのローエンドまたは、ミドルレンジの機種で最多シェアのOSバージョン
- （B）パンチホール有の機種の中で最大のアスペクト比率
- （C）[配信対象OSバージョン](../../requirements/non-functional/usability#配信対象osバージョン)の範囲内で最大の画面解像度
- （D）[配信対象OSバージョン](../../requirements/non-functional/usability#配信対象osバージョン)の範囲内で最小の画面解像度

（※2）[配信対象OSバージョン](../../requirements/non-functional/usability#配信対象osバージョン)の範囲内の最大とノッチ有の画面解像度が重複しているため統一して選定。
  
#### ネイティブ機能

このアプリが利用するネイティブ機能を利用した部分に問題がないことを検証します。
テストに関する事項の[構成テスト（ネイティブ機能バリエーション）](../../requirements/non-functional/test-outline#構成テストネイティブ機能バリエーション)のとおり、選定したネイティブ機能は以下です。

カメラ、生体認証（指紋認証、顔認証）
