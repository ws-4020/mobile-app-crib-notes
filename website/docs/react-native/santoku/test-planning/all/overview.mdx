テストにおける基本方針は以下のとおりです。

## テスト方針

- リスク

  それぞれのテストで必要な端末機種とOSのバリエーションの準備に時間が掛かるため、早期に調整を実施して準備を進めます。

- 品質

  各画面および、機能毎の品質は単体テストにて担保します。結合テストでは画面および、機能と連携するサービスを含めた網羅的なテストと性能テストを実施してアプリ全体の品質を担保します。

テストにおける前提、制約事項は以下のとおりです。

### テスト対象機種とOS

[テスト実施環境](../../requirements/non-functional/test-outline#テスト実施環境)で定義した端末スペックに基づいて、機種とOSバージョンを選定します。
最新の端末および、OSバージョンは、今から約半年前時点で最新のものを対象[^1]とします。
最後にタブレット端末はこのアプリの主要なターゲットにしていないため、実機の検証対象から除外しています。
端末機種とOSバージョンは`2021/10/25`時点で調査したGoogle社とApple社が公開している情報を元に選定します。

[^1]: Google社および、Apple社から発表されているシェア情報が最新のもので約半年前のものであるため、それ以前のものを対象とします。
