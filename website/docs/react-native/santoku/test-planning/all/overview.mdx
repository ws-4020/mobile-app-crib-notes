テストにおける基本方針は以下のとおりです。

## テスト方針

- リスク

  それぞれのテストで必要な端末機種とOSのバリエーションの準備に時間が掛かるため、早期に調整を実施して準備を進めます。

- 品質

  各画面および、機能毎の品質は単体テストにて担保します。結合テストでは画面および、機能と連携するサービスを含めた網羅的なテストと性能テストを実施してアプリ全体の品質を担保します。

テストにおける前提事項と制約事項は、以下のとおりです。

### テスト工程と実施環境

実施するテスト工程とそれぞれの工程毎にどのようなテストを実施するかをまとめています。
また、それぞれのテストで必要なテスト環境についてもまとめています。
テスト実施環境で定義している機種とOSバージョンは、[動作検証端末について](../../requirements/non-functional/performance#動作検証端末について)で定義した選定基準に基づいて、決定します。
