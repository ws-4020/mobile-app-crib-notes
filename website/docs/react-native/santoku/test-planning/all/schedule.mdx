title: 全体スケジュール
---

テストの全体スケジュールを記載します。
各テスト工程内のスケジュールは、各工程のテスト計画書に記載します。

## スケジュール

- スケジュール(矢羽)を引く
  - 見えているサイクル分だけ記載する。後のサイクルは予定が立ってから記載
  - 記載するのはマイルストーンレベル。単体テストがXX完了、結合テストがXX完了、XX時点でテスト完了(後続テストなし)がわかるように
    - 全体の計画に影響が出る場合のみブラッシュアップ。各工程内で遅延やリスケがあっても、全体スケジュールに影響が出ないのであればメンテは不要。
    - 各工程内のブレイクダウンは個別設計書に記載
