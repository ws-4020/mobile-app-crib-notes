理由)

- 開発は全てサンプルアプリチームで行い、他のチームや企業が存在しないので、構成図を作るほどでもない
- 公開物なので、実名が出せない。だとするとあんまり意味ない。。

一般的には、以下を記載する。

## 体制

- 体制図を貼る
   - テスト実施者、管理者(開発チームとしてひとまとめでもOK)
   - ステークホルダがいる場合はわかるようにする(テスト中に仕様変更が入った場合に、誰に合意を取らないといけないかなど)
   - テスト実施者と、不具合修正のチームが分かれる場合は、それも記載
   - 品管部隊が別にいる場合は、それも記載

## 担当者ごとの役割

- 単体テストと結合テストそれぞれのテスト毎に作業概要を列挙して、それを誰が作成するのかを明記する。
- テスト実施以外にも管理系や、
（今回は、サンプルアプリということで作成無しでもいいかもしれない）
