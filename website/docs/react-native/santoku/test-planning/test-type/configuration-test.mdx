## 目的

構成テストでは、このアプリのサポートする端末やOSバージョンごとに期待された動作するかを確認します。このテストを実施することにより、サポート範囲内の端末およびOSバージョンでアプリが正常に動作することを保証します。

## OSバリエーション

### テスト観点

- テスト対象のOSバージョンで、OS依存の機能が正常に動作すること

上記観点の確認のため、テスト対象のOSバージョンで[端末やOSに関する観点](functional-testing/terminal-os.mdx)をテストします。

### テスト方法

全体テスト計画の[OSバージョン](../test-all-planning#osバージョン)のバリエーションを踏まえて、テスト端末を選定します。

[テスト端末バリエーション](../test-variation.mdx)の中から、各テスト計画にて決定した実機でテストを実施します。

## レイアウトバリエーション

### テスト観点

- 全ての機種毎のアスペクト比率、画面解像度毎に表示崩れなどがないこと
- パンチホール有無、ノッチ有無の機種毎に表示崩れなどがないこと

上記観点の確認のため、次のテスト観点を対象にしてアスペクト比率、画面解像度でテストを実施します。

- [レイアウトに関する観点](functional-testing/layout-test.mdx)

### テスト方法

全体テスト計画の[アスペクト比率・画面解像度](../test-all-planning#アスペクト比率画面解像度)、[パンチホール有無・ノッチ有無](../test-all-planning#パンチホール有無ノッチ有無)の以下バリエーションを踏まえて、テスト端末を選定します。

- テスト対象とするアスペクト比率、画面解像度
- ノッチあり
- パンチホールあり
- ノッチ／パンチホールなし

[テスト端末バリエーション](../test-variation.mdx)の中から、各テスト計画にて決定したエミュレータ／シミュレータまたは、実機でテストを実施します。

## ネイティブ機能バリエーション

### テスト観点

- ネイティブ機能の有無や種類（カメラ、指紋認証、顔認証）に関係なくアプリが動作すること

上記観点の確認のため、端末やOSに関する観点の[機種依存機能の検証](functional-testing/terminal-os.mdx#機種依存機能の検証)をネイティブ機能の有無、種類毎で実施します。

### テスト方法

全体テスト計画の[ネイティブ機能の有無、種類](../test-all-planning#ネイティブ機能の有無種類)の以下バリエーションを踏まえて、端末を選定します。

- 顔認証
- 指紋認証
- 生体認証なし
  - 生体認証の無効化で代替可

[テスト端末バリエーション](../test-variation.mdx)の中から、各テスト計画にて決定した実機でテストを実施します。
