## 目的

構成テストでは、このアプリのサポートする端末やOSバージョンごとに期待された動作するかを確認します。このテストを実施することにより、サポート範囲内の端末およびOSバージョンでアプリが正常に動作することを保証します。

## OSバリエーション

### テスト観点

- テスト対象の全OSバージョンで、OS依存の機能が正常に動作すること

上記観点の確認のため、テスト対象の全OSバージョンで[端末やOSに関する観点](functional-testing/terminal-os.mdx)をテストします

### テスト方法

[テスト端末バリエーション](../test-variation.mdx)の中から、Android、iOS／iPadOSで全OSバージョンの実機を1機種ずつ選択してテストを実施します。

## レイアウトバリエーション

### テスト観点

- 全ての機種毎のアスペクト比率、画面解像度毎に表示崩れなどがないこと
- パンチホール有無、ノッチ有無の機種毎に表示崩れなどがないこと
  - 上記観点の確認のため、[画面デザインや出力結果の表示に関する観点](functional-testing/screen-output-result.mdx)を全アスペクト比率、画面解像度で実施する

### テスト方法

- [テスト端末バリエーション](../test-variation.mdx)の中から、Android、iOS／iPadOSで全アスペクト比率、画面解像度をエミュレータ／シミュレータで1機種ずつ選択してテストを実施する

## ネイティブ機能バリエーション

### テスト観点

- ネイティブ機能の有無や種類（カメラ、指紋認証、顔認証）に関係なくアプリが動作すること
  - 上記観点の確認のため、端末やOSに関する観点の[機種依存機能の検証](functional-testing/terminal-os.mdx#機種依存機能の検証)をネイティブ機能の有無、種類毎で実施する

### テスト方法

- [テスト端末バリエーション](../test-variation.mdx)の中から、Android、iOS／iPadOSでネイティブ機能の有無、種類毎に実機を1機種ずつ選択してテストを実施する
