## 目的

- クラス、関数に実装された各業務ロジックについて、設計との差異を発見する
- APIにより取得した結果について、設計との差異を発見する
- 画面ごとの実行結果について、設計との差異を発見する
- 複数の処理をまとめて1つの業務的な意味を持つまとまりにする場合、その実行結果について、設計との差異を発見する

## テスト方法

上記観点のテストは以下のとおり、3種類の方法で実施します。
それぞれどのテスト方法で実施するかについては、テスト観点でテスト方法を決定しています。

- (A) 【自動テスト】 Jestを利用して、作成したテストコードで自動テストを実行して確認
- (B) 【手動テスト】 Android、iOS／iPadOS毎にエミュレータ／シミュレータ
- (C) 【手動テスト】 実機（テスト対象機種分）
