---
title: 画面内操作に関する観点
---

画面内での操作結果のテスト観点をまとめています。
それぞれのテスト観点毎の検証方法は、[テスト方法](../functional-testing.mdx#テスト方法)にまとめています。

## 画面内操作（入力項目）

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | テキスト入力 | プレースホルダーに表示されている内容が設計通りであること | - | A |
| 2 | テキスト入力 | 設計で指定されたデフォルトの入力方式（キーボードなど）が適用されていること | - | A |
| 3 | テキスト入力 | 設計で無制限の入力を明示的に許容している場合を除き、最大入力長が設定されていること | - | A |
| 4 | カレンダー入力 | カレンダーからの日付入力が可能な場合、カレンダー上では入力として許容する期間の日付が設計通りの選択肢のみであること | 例： 予約の場合は過去日は選択できないなど | A |
| 5 | セレクトボックス | デフォルトの選択状態が設計通りであること | - | A |
| 6 | セレクトボックス | 表示される選択肢が設計通りの選択肢のみであること | - | A |
| 7 | ラジオボタン | デフォルトの選択状態が設計通りであること | - | A |
| 8 | ラジオボタン | ラジオボタンについて、設計通りの制御がされていること | 例）複数の中から選択できるものが1つのみとなっているなど | A |
| 9 | チェックボックス | デフォルトのチェック状態が設計通りであること | - | A |
| 10 | ボタン | ボタンについて、設計通りの制御がされていること | 例）連続でのボタン押下ができないようになっているなど | A |
