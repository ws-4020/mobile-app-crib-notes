---
title: ログ出力に関する観点
---

アプリから送信するクラッシュログやユーザ操作ログのテスト観点をまとめています。
それぞれのテスト観点毎の検証方法は、[テスト方法](../functional-testing.mdx#テスト方法)にまとめています。

## ログ出力

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | ユーザ操作 | ユーザ操作に伴う処理中に正常系のログ出力が設計されている場合、設計通りにログが出力されていること | - | A |
| 2 | ユーザ操作 | ユーザ操作に伴う処理中にエラーが発生した場合、[エラーハンドリング](../../../application-architecture/error-handling/overview.mdx)の方針に沿ってログが出力されていること | - | A |
| 3 | ユーザ操作  | ユーザ操作に伴う処理中に正常系のログ出力が設計されている場合、出力されているログが想定どおりの内容であること | - | B |
| 4 | ユーザ操作以外のイベント処理  | 初期データ読み込みなどの処理中に正常系のログ出力が設計されている場合、設計通りにログが出力されていること | - | A |
| 5 | ユーザ操作以外のイベント処理  | 初期データ読み込みなどの処理中にエラーが発生した場合、[エラーハンドリング](../../../application-architecture/error-handling/overview.mdx)の方針に沿ってログが出力されていること | - | A |

## ログ編集

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | 編集仕様 | 出力された正常時のログの内容が設計通りであること | - | A |
| 2 | 編集仕様 | 取得した値を変換する場合、出力された値が設計通りであること | - | A |
| 3 | 編集仕様 | 取得した値を編集する処理が分岐する場合、出力された値が設計通りであること | - | A |
| 4 | 編集仕様 | ループ処理によって、取得した値の編集を繰り返す場合、出力された値が設計通りであること | - | A |
| 5 | 編集仕様 | 取得した値を使用して計算する場合、出力された計算結果の値が設計通りであること | - | A |
| 6 | 編集仕様 | 取得した値を連結する場合、出力された連結された値が設計通りであること | - | A |
| 7 | 編集仕様 | 取得した値を暗号化する設計となっている場合、出力された値が暗号化されること | - | B |
| 8 | 編集仕様 | 取得した値をマスキングする設計となっている場合、出力された値がマスキングされること | - | B |
