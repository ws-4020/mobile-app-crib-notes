---
title: ログ出力に関する観点
---

アプリから送信するクラッシュログやユーザ操作ログのテスト観点をまとめています。
それぞれのテスト観点毎の検証方法は、[テスト方法](../functional-testing.mdx#テスト方法)にまとめています。

## ログ出力

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | 出力内容 | 出力されたログの内容が仕様通りであること | - | A |
| 2 | ユーザ操作 | ユーザ操作に伴う処理中に正常系のログ出力が設計されている場合、仕様通りにログが出力されていること | - | A |
| 3 | ユーザ操作 | ユーザ操作に伴う処理中にエラーが発生した場合、仕様通りにログが出力されていること | - | A |
| 4 | ユーザ操作以外のイベント処理 | 初期データ読み込みなどの処理中に正常系のログ出力が設計されている場合、仕様通りにログが出力されていること | - | A |
| 5 | ユーザ操作以外のイベント処理 | 初期データ読み込みなどの処理中にエラーが発生した場合、仕様通りにログが出力されていること | - | A |
| 6 | セキュリティ | 機密情報について、元の情報が特定できない状態になっていること | - | A |
| 7 | セキュリティ | 個人情報について、元の情報が特定できない状態になっていること | - | A |
