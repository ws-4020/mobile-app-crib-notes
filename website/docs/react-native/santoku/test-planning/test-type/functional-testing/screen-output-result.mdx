---
title: 画面デザインや出力結果の表示に関する観点
---

機能テストに関する画面デザインや出力結果の表示のテスト観点をまとめています。
それぞれのテスト観点毎の検証方法は、[テスト方法](../functional-testing.mdx#テスト方法)にまとめています。

## 画面全体のデザイン

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | デザイン | ロード中にローディングインジケータが表示される場合、表示結果が設計通りであること | - | B |
| 2 | デザイン | ロード完了後の画面のレイアウトがデザイン通りであり、崩れていないこと | - | B |
| 3 | デザイン | スクロール可能な領域が設計通りであること | 例えば、ページの一部の領域のみをスクロールするべきなのに、ページ全体がスクロールされていないかなど | B |

## UI部品

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | データ | UI部品で表示するデフォルト値が設計通りであること | - | B |
| 2 | データ | UI部品で表示する項目の取得元が設計通りであること | - | B |
| 3 | データ | UI部品で表示する項目の編集仕様が設計通りであること | - | B |
| 4 | データ | UI部品で表示する項目のソート順が設計通りであること | 例：プルダウンやラジオボタンなど | B |
| 5 | 活性/非活性 | 操作を受け付けているUI部品が活性表示されていること | - | B |
| 6 | 活性/非活性 | 操作を受け付けていないUI部品が非活性表示されていること | - | B |
| 7 | 活性/非活性 | 活性状態が切り替わる条件を満たした時に、活性・非活性状態に応じて表示が切り替わること | - | B |
| 8 | 表示/非表示 | 表示/非表示が切り替わるUI部品のデフォルトの表示状態が設計通りであること | - | B |
| 9 | 表示/非表示 | 表示/非表示が切り替わる条件を満たした時に、表示状態が切り替わること | - | B |

## 表示項目

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | テキスト表示（固定文字列） | 設計通りの文章が表示されていること | - | B |
| 2 | テキスト表示（外部データ） | 設計通りの文章が表示されていること（別の外部データが誤って表示されていないこと） | - | B |
| 3 | テキスト表示（外部データ） | 入力データの文字列長が長い場合にもレイアウトが崩れないこと | - | B |
| 4 | テキスト表示（外部データ） | 入力データに含まれる改行と同じ位置で画面上でも改行されて表示されていること | - | B |
| 5 | テキスト表示（外部データ） | サロゲートペア文字を許容する場合、サロゲートペア文字が表示されていること | 絵文字などを表示する | B |
| 6 | テキスト表示（外部データ） | 入力データをマスキングして表示する設計の場合、マスキングされた値が表示されていること | - | B |
| 7 | テキスト表示 | 表示内容が数値である場合、設計で指定されたフォーマット通りに表示されていること | - | B |

## 表示項目（一覧）

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | リスト表示 | 表示データの件数が0件の場合のレイアウトが崩れていないこと | - | B |
| 2 | リスト表示 | 表示データの件数が1件の場合のレイアウトが崩れていないこと | - | B |
| 3 | リスト表示 | 表示データの件数が上限件数の場合のレイアウトが崩れていないこと | - | B |
| 4 | リスト表示 | デフォルトのソート順が設計で定められている場合、設計通りのソート順で表示されていること | - | B |
| 5 | リスト表示 | 表示されたデータのソート順を変更する機能が存在する場合、実行すると指定されたソート順にデータがソートされて表示されること | - | B |
| 6 | リスト表示 | 表示データの件数が一覧の表示可能件数を超えた場合、ページングまたはスクロールによる自動追加読込み（無限スクロール）等によって続きが表示できること | - | B |
| 7 | リスト表示 | 表示データの取得に失敗した場合、設計通りの内容が表示されること | - | B |

## 入力項目

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | テキスト入力 | テキスト入力欄に対応するラベルをタップすると、テキスト入力欄にフォーカスが移動すること | - | B |
| 2 | テキスト入力 | パスワード等の機密情報の入力欄は入力内容が非表示になっていること | - | B |
| 3 | テキスト入力 | 設計で定められた最大長まで入力された状態でもレイアウト崩れがないこと | - | B |
| 4 | テキスト入力 | 入力完了後のフォーカス移動が設計で定められている場合、入力後に設計通りフォーカスが移動すること | - | B |
| 5 | テキスト入力 | 現在の入力文字数を表示する場合、サロゲートペア文字や異体字を含む文字列を入力しても正しい入力文字数が表示されること | - | B |
| 6 | テキスト入力 | テキスト入力の要素ごとに、設計通りのキーボード種別で表示されていること | - | B |
| 7 | テキスト入力 | 入力済みの項目をクリアする仕様がある場合、クリアを実行することで対象の項目がクリアされていること | - | B |
| 8 | セレクトボックス | 表示される選択肢が動的に変化する場合、変化する条件に応じて設計通りの選択肢のみが表示されていること | - | B |
| 9 | ファイルアップロード | ファイルアップロードするUI部品は、アップロード中のステータスが分かるようなUIを表示していること | 例：プログレスバー | B |
| 10 | ファイルアップロード | ファイルアップロードするUI部品は、アップロードが完了したらファイル名や画像が正しく表示されていること | - | B |

## 操作項目

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | ボタン | ボタンは、短時間にダブルタップしても押下時の処理が多重実行されないよう制御されていること（再実行可能になるまでの間は非活性となり再度押せないよう制御されていること） | - | B |
| 2 | ボタン | 非同期処理を実行するボタンの場合は、ボタンを押すと非活性状態となりローディングインジケータが表示されること | 例えば、ボタンを押下してHTTP APIの処理を実行し、処理が終了するまでボタンが非活性となっていることなど | B |

## アニメーション

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | アニメーション | 画面遷移時のアニメーションがデザイン通りであること | - | B |
| 2 | アニメーション | ダイアログやモーダルウィンドウ表示時のアニメーションがデザイン通りであること | - | B |
| 3 | アニメーション | リスト表示にアイテムを追加・削除した場合のアニメーションがデザイン通りであること | - | B |

## エラー

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | デザイン | エラー発生時の画面のレイアウトがデザイン通りであり、崩れていないこと | - | B |
| 2 | メッセージ | エラー発生時のエラーメッセージが設計通りの内容となっていること | - | B |
| 3 | メッセージ | 単項目バリデーションの結果、エラー条件に該当した場合、該当する箇所へ設計通りのエラーメッセージが表示されていること | - | B |
| 4 | メッセージ | 相関バリデーションの結果、エラー条件に該当した場合、該当する箇所へ設計通りのエラーメッセージが表示されていること | - | B |
| 5 | メッセージ | タイムアウトした場合、設計通りのエラーメッセージが表示されていること | - | B |
