title: ユースケースに関する観点
---

このアプリの利用シナリオをベースに網羅的なテストを実施します。
以下より、テストとして必要なテストシナリオを詳細化して実施するテスト内容を決定します。
それぞれのユースケース毎の検証方法は、[テスト方法](../functional-testing.mdx#テスト方法)にまとめています。

| NO | ユースケース | 内容 | 例 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | 事前条件 | このアプリの基本となる利用フローを検証する条件 | 利用者が既にこのアプリをダウンロード済であり、アカウントも作成済 | B、C |
| 2 | トリガー | それぞれのユースケースを開始するための条件 | チームへ参加する招待コードの受け取り | B、C |
| 3 | 基本フロー | 利用シナリオを踏まえた正常系ルートの動作 | 時間割テンプレートを作成（編集含む）し、当日の時間割を作成して、時間割を開始 | B、C |
| 4 | 代替フロー | 例外系の動作 | チームへ招待する招待コードの有効期限切れ | B、C |
| 5 | 事後条件 | 期待する結果 | 当日の時間割を開始して、このアプリの利用が開始できていること | B、C |
