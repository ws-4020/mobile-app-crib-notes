---
title: 構成テスト
---

構成テストに関連するテスト観点をまとめています。

- 構成テストは、OS/レイアウト/ネイティブ機能の各バリエーションごとに実施するため、それぞれのテスト観点を定義します。
- 構成テストは、複数の端末で実施します。端末の選定基準は、各バリエーションのテスト対象端末をご参照ください。
- テスト観点は、[機能テストに関するテスト観点](function-test.mdx#機能テストに関するテスト観点)より抜粋します。

## OSバリエーションテスト

Android、iOS／iPadOS、それぞれのテスト対象OSバージョンを対象にOSバージョンが相違しても問題なく動作することを検証します。
テスト対象端末とテスト観点は以下のとおりです。

### テスト対象端末：OSバリエーション

[テスト端末バリエーション](/react-native/santoku/test-planning/test-variation.mdx)の中から、
Android、iOS／iPadOSでそれぞれテスト対象OSバージョンごとに1端末ずつ選定します。

### テスト観点：OSバリエーション

- [端末やOSに関する観点](/react-native/santoku/test-planning/test-point/function-test/terminal-os.mdx)（※1）

（※1）[端末やOSに関する観点](/react-native/santoku/test-planning/test-point/function-test/terminal-os.mdx)の中から、機種依存機能の検証を除く観点がテスト対象観点になります。

## レイアウトバリエーションテスト

スマートフォンの機種は多種多様であり、それに伴って画面サイズのバリエーションが豊富に存在します。
ここでは、それぞれでテスト対象としたアスペクト比率、画面解像度を対象に表示崩れなどがないことを検証します。
テスト対象端末とテスト観点は以下のとおりです。

### テスト対象端末：レイアウトバリエーション

[テスト端末バリエーション](/react-native/santoku/test-planning/test-variation.mdx)の中から、
Android、iOS／iPadOSでそれぞれテスト対象のアスペクト比率、画面解像度ごとに1端末ずつ選定します。

### テスト観点：レイアウトバリエーション

- [画面デザインや出力結果の表示に関する観点](/react-native/santoku/test-planning/test-point/function-test/screen-output-result.mdx)

## ネイティブ機能バリエーションテスト

このアプリに関連するネイティブ機能を対象にネイティブ機能の有無や種別に関係なく動作することを検証します。
テスト対象端末とテスト観点は以下のとおりです。

### テスト対象端末：ネイティブ機能バリエーション

[テスト端末バリエーション](/react-native/santoku/test-planning/test-variation.mdx)の中でネイティブ機能ごとに1端末ずつ選定します。

### テスト観点：ネイティブ機能バリエーション

- 端末やOSに関する観点の[機種依存機能の検証](/react-native/santoku/test-planning/test-point/function-test/terminal-os.mdx#機種依存機能の検証)
