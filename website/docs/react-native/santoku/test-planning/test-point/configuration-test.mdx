---
title: 構成テスト
---

構成テストに関連するテスト観点をまとめています。

- 構成テストは、OS/レイアウト/ネイティブ機能の各バリエーションごとに実施するため、それぞれのテスト観点を定義します。
- 構成テストは、複数の端末で実施します。端末の選定基準は、各バリエーションのテスト対象端末をご参照ください。
- テスト観点は、[機能テストに関するテスト観点](function-test.mdx#機能テストに関するテスト観点)より抜粋します。

## OSバリエーション

### テスト対象端末

[テスト端末バリエーション](/react-native/santoku/test-planning/test-variation.mdx)の中から、
Android、iOS(iPadOS)でそれぞれOSバージョンごとに1端末ずつ選定します。

### テスト観点

- [端末やOSに関する観点](/react-native/santoku/test-planning/test-point/function-test/terminal-os.mdx)

## レイアウトバリエーション

### テスト対象端末

[テスト端末バリエーション](/react-native/santoku/test-planning/test-variation.mdx)の中から、
Android、iOS(iPadOS)でそれぞれ端末のアスペクト比率と画面解像度ごとに1端末ずつ選定します。
ただし、同じアスペクト比率または画面解像度であっても、パンチホール/ノッチの有無が分かれる場合は、どちらも対象端末とします。

:::note
上記の観点で端末を選定すると、今回はテスト端末バリエーションの全端末が対象となります。
:::
