---
title: イベント実行時のエラーに関する観点
---

機能テストに関するイベント実行時のエラーのテスト観点をまとめています。
それぞれのテスト観点の実施方法の種別を以下に列挙しています。

- (A) [自動テスト]Jestを利用して、作成したテストコードで自動テストを実行して確認
- (B) [手動テスト]Android、iOS／iPadOS毎にエミュレータ／シミュレータ
- (C) [手動テスト]実機（テスト対象機種分）

## エラーハンドリング

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | ユーザ操作 | ユーザ操作に伴う処理中にエラーが発生した場合、エラーの内容や取るべきアクションがユーザに伝わるように何らかの形で通知されていること | - | A |
| 2 | ユーザ操作 | ユーザ操作に伴う処理中にエラーが発生した場合、再実行が可能な場合はボタン等の非活性状態が解除されていること | - | A |
| 3 | ユーザ操作以外のイベント処理 | 初期データ読み込みなどの処理中にエラーが発生した場合、リトライ可能な処理であればリトライ可能なUIが提供されていること（一覧に表示する情報を取得するのに失敗した場合、WebViewに表示するページの取得に失敗した場合など） | - | A |
| 4 | ユーザ操作以外のイベント処理 | 初期データ読み込みなどの処理中にエラーが発生した場合、ユーザ影響がある場合はエラーの内容や取るべきアクションがユーザに伝わるように何らかの形で通知されていること | - | A |
