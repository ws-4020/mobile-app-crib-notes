---
title: 画面デザイン
---

機能テストに関する画面デザインのテスト観点をまとめています。
それぞれのテスト観点の実施方法の種別を以下に列挙しています。

- (A) [自動テスト]Jestを利用して、作成したテストコードで自動テストを実行して確認
- (B) [手動テスト]Android、iOS／iPadOS毎にエミュレータ／シミュレータ
- (C) [手動テスト]実機（テスト対象機種分）

## 画面デザイン

### 画面全体のレイアウト

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | デザイン | ロード中にローディングインジケータが表示されていること | - | B |
| 2 | デザイン | ロード完了後の画面のレイアウトがデザイン通りであり、崩れていないこと | - | B |
| 3 | デザイン | エラー発生時の画面のレイアウトがデザイン通りであり、崩れていないこと | - | B |
| 4 | デザイン | スクロール可能な領域が設計通りであること | 例えば、ページの一部の領域のみをスクロールするべきなのに、ページ全体がスクロールされていないかなど | B |

### アニメーション

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | アニメーション | 画面遷移時のアニメーションがデザイン通りであること | - | B |
| 2 | アニメーション | ダイアログやモーダルウィンドウ表示時のアニメーションがデザイン通りであること | - | B |
| 3 | アニメーション | リスト表示にアイテムを追加・削除した場合のアニメーションがデザイン通りであること | - | B |

### 表示項目

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | テキスト表示（固定文字列） | 設計通りの文章が表示されていること | - | B |
| 3 | テキスト表示（外部データ） | 表示されている文章が設計通りであること（別のデータが誤って表示されていないこと） | - | B |
| 4 | テキスト表示（外部データ） | 入力データの文字列長が長い場合にもレイアウトが崩れないこと | - | B |
| 5 | テキスト表示（外部データ） | 入力データに含まれる改行と同じ位置で画面上でも改行されて表示されていること | - | B |
| 6 | テキスト表示（外部データ） | サロゲートペア文字を許容する場合、サロゲートペア文字を含む値を表示した結果がエラーとならないこと | 絵文字などを表示する | B |
| 7 | テキスト表示（外部データ） | 入力データをマスキングして表示する設計の場合、マスキングされた値が表示されていること | - | B |
| 10 | テキスト表示（数値） | 表示内容が数値である場合、設計で有効桁数が指定されている場合はその有効桁数通りに表示されていること | - | B |
| 11 | テキスト表示（数値） | 表示内容が数値である場合、設計で3桁毎のカンマ区切りでの表示が指定されている場合はその通りに表示されていること | - | B |
| 12 | リスト表示 | 表示データの件数が0件の場合に表示される内容が設計通りであること | - | B |
| 13 | リスト表示 | 表示データの件数が1件の場合のレイアウトが崩れていないこと | - | B |
| 14 | リスト表示 | 表示データの件数が上限件数の場合のレイアウトが崩れていないこと | - | B |
| 15 | リスト表示 | 表示データの取得に失敗した場合に表示される内容が設計通りであること | - | B |
| 16 | リスト表示 | デフォルトのソート順が設計で定められている場合、設計通りのソート順で表示されていること | - | B |
| 17 | リスト表示 | 表示されたデータのソート順を変更する機能が存在する場合、実行すると指定されたソート順にデータがソートされて表示されること | - | B |
| 18 | リスト表示 | 一覧の表示件数が上限を超えた場合、ページングまたはスクロールによる自動追加読込み（無限スクロール）等によって続きが表示できること | - | B |
| 19 | リスト表示 | Pull-to-refreshによって画面表示している情報を最新化できること | - | B |

### 入力項目

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | テキスト入力 | プレースホルダーに表示されている内容が設計通りであること | - | B |
| 2 | テキスト入力 | 設計で指定されたデフォルトの入力方式（ひらがな入力など）が適用されていること | - | B |
| 3 | テキスト入力 | テキスト入力欄に対応するラベルをタップすると、テキスト入力欄にフォーカスが移動すること | - | B |
| 4 | テキスト入力 | パスワード等の機密情報の入力欄は入力内容が非表示になっていること | - | B |
| 5 | テキスト入力 | 設計で無制限の入力を明示的に許容している場合を除き、最大入力長が設定されていること | - | B |
| 6 | テキスト入力 | 設計で定められた最大長まで入力された状態でもレイアウト崩れがないこと | - | B |
| 7 | テキスト入力 | 入力完了後のフォーカス移動が設計で定められている場合、入力後に設計通りフォーカスが移動すること | - | B |
| 8 | テキスト入力 | 現在の入力文字数を表示する場合、サロゲートペア文字や異体字を含む文字列を入力しても正しい入力文字数が表示されること | - | B |
| 9 | セレクトボックス | デフォルトの選択状態が設計通りであること | - | B |
| 10 | セレクトボックス | 表示される選択肢が設計通りの選択肢のみであること | - | B |
| 11 | セレクトボックス | 表示される選択肢が動的に変化する場合、変化する条件に応じて設計通りの選択肢のみが表示されていること | - | B |
| 12 | ラジオボタン | デフォルトの選択状態が設計通りであること | - | B |
| 13 | チェックボックス | デフォルトのチェック状態が設計通りであること | - | B |
| 14 | ファイルアップロード | ファイルアップロードするUI部品は、アップロード中のステータスが分かるようなUIを表示していること | 例：プログレスバー | B |
| 15 | ファイルアップロード | ファイルアップロードするUI部品は、アップロードが完了したらファイル名や画像が正しく表示されていること | - | B |

### 操作項目

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | ボタン | ボタンは、短時間にダブルタップしても押下時の処理が多重実行されないよう制御されていること | - | B |
| 2 | ボタン | 多重に実行されると問題のある処理を行うボタンは、一度実行されると再実行可能になるまでの間は非活性となり再度押せないよう制御されていること | - | B |
| 3 | ボタン | 非同期処理を行うボタンの場合は、ボタンを押すと非活性状態となりローディングインジケータが表示されること | - | B |
| 4 | ボタン | 破壊的な操作を行うボタンの場合は、ボタンを押すと処理を開始する前にユーザへの確認が行われること | 例：削除ボタン | B |

### UI部品の状態変化

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | 活性/非活性 | 操作を受け付けているUI部品が活性表示されていること | - | B |
| 2 | 活性/非活性 | 操作を受け付けていないUI部品が非活性表示されていること | - | B |
| 3 | 活性/非活性 | 活性状態が切り替わる条件を満たした時に、活性・非活性状態に応じて表示が切り替わること | - | B |
| 4 | 表示/非表示 | 表示/非表示が切り替わるUI部品のデフォルトの表示状態が設計通りであること | - | B |
| 5 | 表示/非表示 | 表示/非表示が切り替わる条件を満たした時に、表示状態が切り替わること | - | B |
