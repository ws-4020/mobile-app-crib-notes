## 機能テスト

- (A) [自動テスト]Jestを利用して、作成したテストコードで自動テストを実行して確認
- (B) [手動テスト]Android、iOS／iPadOS毎にエミュレータ／シミュレータ
- (C) [手動テスト]実機（テスト対象機種分）

## 全般

単体/結合で全般的な観点
| 8 | テキスト表示（導出データ） | 演算や文字列結合などにより導出した値を表示する場合、表示された値が設計されたロジック通りであること | - | A |
| 9 | テキスト表示（導出データ） | 演算によって導出した値を表示する場合、オーバーフローが発生しないこと | - | A |
| 5 | ボタン | 外部へのデータ送信を行う場合は、送信されるデータが設計通りであること | コードの変換、トリム処理、文字列の一部を取得する処理なども検証する | A |
| 1 | ストレージ | 端末のストレージ内にファイルを保存する場合、設計通りにファイルが保存されていないこと | - | B |
| 1 | メモリリーク | 複数回縦横切り替えを行ってもメモリリークしないこと | 画面切り替え時の初期化処理でメモリリークする可能性があるため | B |
| 2 | メモリリーク | 画像ファイル等の大きなサイズを画面間で持ちまわす場合に、メモリリークしないこと | - | B |
| 1 | 要求権限 | アプリが使用する権限が要求されていること | - | B |
