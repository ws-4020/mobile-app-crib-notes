機能テストに関連するそれぞれのテスト観点をまとめています。

## 機能テスト全般のテスト観点

機能テスト全般として、要求事項を満たしていることを検証するテストを実施します。
具体的な観点としては、以下のものになります。

- 各機能のロジックについて、設計との差異がないこと
- 画面単位で、レイアウトの表示結果や入力項目、出力項目の結果が設計と差異がないこと
- 複数の処理をまとめて1つの業務的な意味を持つまとまりにする場合、その実行結果について、設計との差異を発見する
- インターフェースファイルの出力結果について、設計との差異を発見する

## ユースケースの観点

- 事前条件
- トリガ
- 基本フロー
- 代替フロー
- 例外フロー
- 事後条件
