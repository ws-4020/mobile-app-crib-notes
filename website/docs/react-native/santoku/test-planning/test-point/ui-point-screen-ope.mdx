---
title: 画面操作を確認する観点
---

ここでは単体テストの画面操作を確認するテスト観点をまとめています。
それぞれのテスト観点を検証する際のテスト方法の種類は、以下のとおりです。

- （A）Jestを利用して、作成したテストコードで自動テストを実行して確認
- （B）シミュレータ（1機種×OS）
- （C）シミュレータ（アスペクト比が違う複数パターン）
- （D）実機（テスト対象端末の中から最も性能が低いもの×OS）
- （E）実機（テスト対象端末分）

## イベント実行結果

### テスト項目：UI部品

| NO | 詳細 | 観点 | 補足 | テスト方法 |
| --- | --- | --- | --- | --- |
| 1 | - | ボタンは、二度押し防止の制御がされていること | - | B |
| 2 | - | テキスト入力が可能なUI部品は、デフォルトで指定された入力方式（ひらがな入力など）が設計通りであること | - | B |
| 3 | - | テキスト入力が可能なUI部品に最大長を入力した場合、レイアウト崩れがないこと | - | C |
| 4 | - | 画面表示している情報を最新化するために、Pull-to-refreshができること | - | B |
| 5 | - | 一覧の表示件数が上限を超えた場合、以下の対応が実施できること（ページング、スクロールで追加読込み（無限スクロール） | - | B |
| 6 | - | ファイルアップロードするUI部品は、アップロード中のステータスが分かるようなUIを表示していること | 例：プログレスバー | B |

### テスト項目：エラー

| NO | 詳細 | 観点 | 補足 | テスト方法 |

| 2 | - | エラーが発生した場合、ログの出力が設計通りになっていること | - | A |
