機能テストに関連するそれぞれのテスト観点をまとめています。

## 機能テスト全般のテスト観点

機能テスト全般として、要求事項を満たしていることを検証するテストを実施します。
具体的な観点としては、以下のものになります。

- 各機能のロジックについて、設計との差異がないこと
- 画面単位で、レイアウトの表示結果や入力項目、出力項目の結果が設計と差異がないこと
- 複数の処理をまとめて1つの業務的な意味を持つまとまりにする場合、その実行結果について、設計との差異を発見する
- インターフェースファイルの出力結果について、設計との差異を発見する

## ユースケースの観点

このアプリの利用シナリオをベースに検証する網羅的なテストを実施します。
以下の観点より、テストとして必要なユースケースの観点を詳細化して決定します。

| ユースケース | 内容 | 例 |
| :--- | :--- | :--- |
| 事前条件 | このアプリの基本利用をフローを検証する条件 | 利用者が既にこのアプリをダウンロード済であり、アカウントも作成済 |
| トリガー | それぞれのユースケースを開始するための条件 | チームへ参加する招待コードを受け取り |
| 基本フロー | 利用シナリオを踏まえて正常系ルートの動作 | 時間割テンプレートを作成（編集含む）して、当日の時間割を作成して、時間割を開始 |
| 代替フロー | 例外系の動作 | チームへ招待する招待コードの有効期限切れ |
| 事後条件 | 期待する結果 | 当日の時間割を開始して、このアプリの利用が開始できていること |
