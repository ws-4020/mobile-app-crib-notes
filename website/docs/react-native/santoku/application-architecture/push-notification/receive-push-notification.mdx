## プッシュ通知受信時の処理

処理フロー図とそれぞれの処理の説明。

![プッシュ通知受信時の処理フロー](receive-push-notification.drawio.png)

- デバイスの状態によりPush通知の取り扱いが変わる
  - Foreground
    - アプリケーションが起動して表示中
  - Background
    - アプリは起動中だがバックグラウンドにいる
  - Quit
    - デバイスがロックされている、もしくはアプリがアクティブでないか実行されていないとき

- isHeadlessの説明は必要？
